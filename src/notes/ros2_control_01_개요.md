# ROS2 Control - 01. 개요

## ROS2 Control이란?

ROS2 Control은 로봇의 하드웨어(모터, 센서 등)를 **표준화된 방식으로 제어하기 위한 프레임워크**입니다.

---

## 왜 필요한가?

### ROS2 Control이 없다면

로봇마다 제조사가 다르고, 모터 드라이버도 다릅니다.
Nav2나 MoveIt 같은 상위 소프트웨어는 "이 로봇이 어떤 모터를 쓰는지" 알 필요 없이 그냥 속도 명령만 내리고 싶습니다.

그런데 ROS2 Control이 없다면:

```
Nav2 → "이 로봇은 Dynamixel이니까 Dynamixel SDK로 직접 제어해야지"
Nav2 → "저 로봇은 다른 드라이버니까 코드를 다시 짜야지"
```

상위 소프트웨어가 하드웨어에 종속됩니다.

### ROS2 Control이 있다면

```
Nav2 → /cmd_vel 토픽으로 속도 명령 발행
             ↓
       ROS2 Control이 알아서 해당 하드웨어에 맞게 변환
             ↓
       Dynamixel이든, 다른 드라이버든 상관없이 동작
```

상위 소프트웨어는 하드웨어를 몰라도 됩니다.

---

## 핵심 개념 - 콘센트와 플러그 비유

ROS2 Control을 이해하는 가장 좋은 비유는 **콘센트(표준 인터페이스)** 입니다.

```
[벽의 콘센트]  ←→  [가전제품 플러그]
  (표준 규격)        (제품마다 다르지만 규격은 같음)
```

ROS2 Control에서는:

```
[command_interface / state_interface]  ←→  [컨트롤러]
         (표준 인터페이스)                    (Nav2, MoveIt, 직접 작성 등)
```

- **콘센트(인터페이스)**: 하드웨어가 제공하는 표준 창구 (속도 명령 받기, 위치 읽기 등)
- **플러그(컨트롤러)**: 인터페이스를 통해 하드웨어를 사용하는 소프트웨어
- **콘센트 규격을 맞추면**: 어떤 플러그(컨트롤러)든 꽂아서 사용 가능

## ROS2 Control의 3개 레이어

```
┌─────────────────────────────────────────┐
│            상위 소프트웨어               │
│   (Nav2, MoveIt, teleop, 직접 작성...)  │
│         /cmd_vel 토픽으로 명령           │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│             Controller                  │
│  (diff_drive_controller, PID 등)        │
│  토픽 명령을 받아 interface 값으로 변환  │
└────────────────┬────────────────────────┘
                 │  command/state interface (내부 메모리)
┌────────────────▼────────────────────────┐
│         Controller Manager              │
│  하드웨어와 컨트롤러를 연결하는 중간 관리자│
│  제어 루프(read → update → write) 실행  │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│         Hardware Interface              │
│  (실제 드라이버 또는 mock_components)   │
│  실제 모터/센서와 통신                   │
└─────────────────────────────────────────┘
```

---

## 실제 로봇 vs 시뮬레이션

| 구분 | 하드웨어 플러그인 | 설명 |
|---|---|---|
| 학습/개발 | `mock_components/GenericSystem` | 실제 하드웨어 없이 소프트웨어만으로 동작 |
| Gazebo 시뮬레이션 | `gazebo_ros2_control/GazeboSystem` | Gazebo 물리 엔진과 연결 |
| 실제 로봇 | `my_robot_hardware/MyRobotHardware` 등 | 실제 모터 드라이버와 통신 |

**핵심**: 하드웨어 플러그인만 교체하면 나머지 코드(컨트롤러, Nav2 등)는 그대로 사용 가능합니다.
